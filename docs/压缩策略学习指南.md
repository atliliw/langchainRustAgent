# 压缩策略学习指南

## 一、5种压缩模式对比

| 模式 | 原理 | 适用场景 |
|------|------|----------|
| **none** | 不压缩，保留全部 | 测试/短对话 |
| **sliding_window** | 只保留最近N条 | 简单场景 |
| **token_limit** | 控制总token数 | 限制成本 |
| **summary** | 全部压缩成摘要 | 长对话 |
| **layered** | 重要+摘要+最近 | **推荐** |

---

## 二、参数修改实验

### 实验1：修改保留消息数

```toml
[conversation]
keep_recent_messages = 10  # 从5改为10
```

**效果**：更多最近对话被保留，上下文更连贯

### 实验2：修改压缩阈值

```toml
[conversation]
compress_threshold = 30  # 从15改为30
```

**效果**：更长对话后才触发压缩，节省API调用

### 实验3：添加重要关键词

```toml
[conversation]
important_keywords = ["我的名字", "我是", "记住", "项目", "需求"]
```

**效果**：包含这些词的消息永不压缩

## 六、推荐配置（生产环境）

```toml
[conversation]
max_history_messages = 50
compress_threshold = 20
keep_recent_messages = 8
important_keywords = ["用户", "项目", "设定", "规则", "需求"]
summary_model = "gpt-3.5-turbo"
```

---

## 七、常见问题

**Q: 压缩后消息丢失？**
A: 添加更多 `important_keywords`

**Q: Token消耗还是太多？**
A: 减小 `max_history_messages` 或增大 `compress_threshold`

**Q: 摘要质量差？**
A: 检查 `summary_model` 是否可用，或改用更好的模型
