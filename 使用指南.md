# LangChainRust Demo 使用指南

## 快速开始

### 1. 打开 Web 界面

浏览器访问: **http://192.168.10.100:8080**

你会看到三个功能标签页:
- **文件上传** - 上传文档进行向量化
- **向量搜索** - 搜索已上传的文档
- **精准度测试** - 测试搜索效果

---

## 功能使用

### 📤 文件上传

1. 点击 **"文件上传"** 标签
2. 点击或拖拽文件到上传区域
3. 支持的文件类型:
   - `.txt` - 纯文本
   - `.pdf` - PDF 文档
   - `.md` - Markdown
   - `.json` - JSON 数据
   - `.csv` - CSV 表格
4. 文件会自动:
   - 分割成小块
   - 生成向量嵌入
   - 存储到 Qdrant

**示例**: 上传一个包含公司文档的 PDF，系统会自动处理并向量化。

---

### 🔍 向量搜索

1. 点击 **"向量搜索"** 标签
2. 在输入框输入查询内容，   - 例如: `"公司的核心业务是什么？"`
   - 例如: `"产品的主要功能"`
3. 可调整返回结果数量 (默认 5)
4. 点击 **"搜索"** 按钮

**返回结果包含**:
- 文档内容片段
- 相似度分数 (百分比)
- 文档元数据 (来源文件、上传时间等)

---

### 📊 琜索精准度测试

1. 点击 **"精准度测试"** 标签
2. 点击 **"运行测试"** 按钮
3. 系统会:
   - 自动插入 5 个测试文档
   - 对每个测试文档运行搜索
   - 计算精准度分数

**测试报告包含**:
- 总测试数 / 通过数
- 精准度分数 (百分比)
- 平均排名位置
- 每个测试的详细结果

## API 端点 (可选)

如果你需要通过代码调用:

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/upload` | POST | 上传文件 (multipart/form-data) |
| `/api/search` | POST | 搜索 `{query, top_k}` |
| `/api/stats` | GET | 获取统计信息 |
| `/api/clear` | POST | 清空所有文档 |
| `/api/test/precision` | POST | 运行精准度测试 |

**示例 - 搜索**:
```bash
curl -X POST http://192.168.10.100:8080/api/search \
  -H "Content-Type: application/json" \
  -d '{"query": "公司业务介绍", "top_k": 3}'
```

## 配置说明

配置文件位于: `/opt/langchainrust/demo/config.toml`

### 重要配置项

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| `openai.api_key` | 已配置 | OpenAI API Key |
| `openai.base_url` | https://api.openai-proxy.org/v1 | API 代理地址 |
| `qdrant.url` | http://localhost:6334 | Qdrant 服务地址 |
| `qdrant.collection_name` | demo_documents | 存储集合名称 |
| `qdrant.vector_size` | 1536 | 向量维度 (匹配 embedding 模型) |

**修改配置后需重启服务**:
```bash
systemctl restart langchainrust-demo
```

## 常用运维命令

```bash
# 查看服务状态
systemctl status langchainrust-demo

# 查看实时日志
journalctl -u langchainrust-demo -f

# 重启服务
systemctl restart langchainrust-demo

# 停止服务
systemctl stop langchainrust-demo

# 查看上传的文档统计
curl http://192.168.10.100:8080/api/stats
```

## 清空数据

如果你想重新测试:

### 方法 1: Web 界面
在"文件上传" 页面点击 **"清空所有文档"** 按钮

### 方法 2: API
```bash
curl -X POST http://192.168.10.100:8080/api/clear
```

### 方法 3: Qdrant Dashboard
访问 http://192.168.10.100:6333/dashboard 手动删除 collection

## 技术架构

```
用户请求 → Axum Web 服务
           ↓
    文档处理 (分割 + 向量化)
           ↓
    OpenAI Embeddings API
           ↓
    Qdrant 向量数据库 (192.168.10.100:6334)
```
